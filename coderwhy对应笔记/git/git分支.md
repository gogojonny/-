# git分支

解决了什么问题？

git分支可以让开发朝着不同的方向走。一个比较适用的场景是，当我们开发一个1.0.0版本的软件并发布，并且继续开发接下来的功能，突然有一天在1.0.0版本中出现重大的bug需要修复，正常做法是退回到1.0.0版本并创建新的分支，并在新的分支上对bug进行修改，然后发布1.0.1版本，最后将修改后的版本与新做的功能分支进行合并

应该如何学习？

了解如何创建分支，如何切换分支，如何创建并切换分支，为何能移动分支。分支在修改老版本bug时的具体操作。如何查看分支，如何删除分支，如何合并分支



分支例如master分支，实际上一直跟随者最后一次提交，与tag不同，tag在创建好后会固定下来，而分支会随着提交而移动。切换分支的方法是使用git checkout 《分支名》，切换分支的原理是移动HEAD，HEAD始终指向当前分支。使用上一条语法就是移动HEAD。

对分支的基本操作思路在分支如何解决问题上具体说过了，操作的语法在ppt上有。



## git 工作流（git flow）

解决了什么问题？

git flow是当前社区中比较流行的git工作流，可以让开发过程变得有序，富有扩展性

应该如何学习？

学习当前社区中流行的git工作流，了解每个分支都是干什么的

![img](https://cdn.nlark.com/yuque/0/2023/png/35074430/1675849513300-0cdf50a4-fbf7-41d5-8651-22ed324f93f5.png)

master：主要用于版本发布

hotfix：热修复，用于修改已上线版本的bug

develop：主要开发用的分支

feature：用于给当前软件添加新特性，如果这个特性有价值就会合并，没有价值就不合并

release：用于在上线之前做测试用，减少bug
